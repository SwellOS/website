Design

Swell Kernel

Monolithic, performance-patched, no modules. All common x86_64 drivers baked in. Every config option documented in-tree.

runit

Simple, fast init. No systemd. Base services only — add what you need. /etc/sv/ directories, sv commands.

sbpm

Swell Blazing Package Manager. Binary packages. Written in Rust. Topological dependency resolution. Optional source builds.

KDE Plasma on Wayland

Modern desktop, fully opt-in. Wayland-native, PipeWire audio. Base system is CLI-only — you choose your desktop.

LFS/BLFS based

Built from Linux From Scratch methodology. Pre-compiled binaries with full source available. Nothing is there by accident.

Rolling + Freeze

Rolling release with sbpm freeze to pin all packages. Update on your terms. 6-month stable snapshots.

Philosophy

"Pre-compiled, verified binaries. Full source available. You choose what runs on your machine."

  • Nothing is installed unless you ask for it
  • Every kernel config option has a documented rationale
  • No bloat — no systemd, no dbus, no PulseAudio in the base system
  • You control updates — sbpm freeze pins everything
  • Simple, auditable, hackable

Package Manager

sbpm -S firefox          # download and install binary package
sbpm -R firefox          # remove package and all its files
sbpm -U                  # upgrade all packages
sbpm -Ss multimedia      # search available packages
sbpm -Syu                # sync repo metadata + upgrade all
sbpm -L                  # list installed packages
sbpm -I firefox          # show package info
sbpm freeze              # pin all installed packages
sbpm unfreeze            # unpin and resume rolling

FAQ

Binary or source?

sbpm installs pre-compiled binary packages by default — fast like any binary distro. Source builds are available for users who want custom compile flags.

Why no kernel modules?

All common x86_64 drivers are compiled directly into the kernel. No module loading, no driver management, no "you need to install firmware" headaches. It just works on modern hardware.

Why runit instead of systemd?

runit is simpler — a few hundred lines of C, reads flat files in /etc/sv/, does one thing well. No udev integration, no journal, no logind. dbus/elogind are pulled in only when a desktop is installed.

Can I use a different desktop?

Yes. The base is CLI-only. KDE Plasma is the default desktop metapackage, but you can install GNOME, Sway, i3, or any other WM/DE via sbpm.

How do I update?

sbpm -Syu syncs the repo metadata and upgrades all out-of-date packages. Use sbpm freeze to pin versions if you want stability.

Is this ready to use?

SwellOS is in early development. See the roadmap for progress toward the v0.1 alpha release.
