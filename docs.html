SwellOS Book

This documentation follows the LFS/BLFS book structure. It is a work in progress — sections are added as components are implemented.

Preface

SwellOS is an independent Linux distribution built from the ground up using the Linux From Scratch methodology. Pre-compiled binary packages with full source available. Nothing is installed unless you choose it.

This book guides you through building, installing, and using SwellOS — from kernel configuration to desktop setup.

Introduction

SwellOS targets modern x86_64 hardware. It uses:

  • The Swell kernel — a monolithic fork of Linux with all common drivers baked in and performance tuning applied
  • runit — a simple, fast init system
  • sbpm — the Swell Blazing Package Manager, a binary package manager written in Rust
  • KDE Plasma on Wayland — the default desktop, fully opt-in
  • Ext4 — the default filesystem

Preparing

System Requirements

  • x86_64 CPU (2015 or later recommended)
  • 2 GB RAM minimum (4 GB recommended for desktop)
  • 10 GB disk space minimum (40 GB for desktop + development)
  • UEFI boot support

Building the Toolchain

The toolchain (GCC, binutils, glibc, linux-api-headers) is built from source during installation. This is done once and cached.

Swell Kernel

The Swell kernel is a fork of Linux with the following design:

  • Monolithic — no loadable kernel module support. All drivers are compiled in.
  • Performance-patched — custom scheduler tuning, preemptible, tickless.
  • All common x86_64 drivers baked in: GPU (Intel/AMD/Nvidia), NVMe, SATA, wired/wireless networking, USB, Ext4/Btrfs/FAT/NTFS, input, audio.
  • No legacy hardware: no IDE, floppy, ISA, 32-bit, ancient drivers.
  • ~4-6 MB uncompressed.

Every kernel config option has an in-tree comment explaining its purpose. The kernel does not need to be rebuilt when hardware changes — all common hardware is already supported.

runit

SwellOS uses pure runit as its init system, with no wrappers.

runit          # PID 1
sv status dhcpcd
sv up dhcpcd
sv down dhcpcd

Service directories live in /etc/sv/. Enabled services are symlinked in /var/service/.

Base services: udevd, dhcpcd, getty@tty1, sshd (optional).

sbpm — Swell Blazing Package Manager

sbpm is the binary package manager for SwellOS, written in Rust.

sbpm -S firefox          # download and install binary package
sbpm -R firefox          # remove
sbpm -U                  # upgrade all packages
sbpm -Ss query           # search
sbpm -Syu                # sync + upgrade
sbpm -L                  # list installed
sbpm -I firefox          # package info
sbpm freeze              # pin all packages
sbpm unfreeze            # unpin

Packages are distributed as pre-compiled .swell archives (tar.zst containing metadata.toml, manifest, and root/). Build scripts are maintained for the official build server and optional local compilation via swell-build.

Core Packages

The core system packages include:

  • dash — /bin/sh
  • glibc — standard C library
  • gcc — C/C++ compiler
  • binutils — assembler, linker, utilities
  • linux-api-headers — kernel headers for userspace
  • runit — init system
  • sbpm — package manager

Desktop

The default desktop is KDE Plasma on Wayland. Installed via the kde-plasma metapackage:

sbpm -S kde-plasma

This pulls in Plasma, Wayland, PipeWire, dbus, elogind, and NetworkManager. The base system is CLI-only — the desktop is entirely opt-in.

Installer

The SwellOS installer (swell-install) is a terminal UI that guides you through:

  1. Partitioning the disk (guided or manual)
  2. Selecting the filesystem (Ext4 default, Btrfs optional)
  3. Selecting metapackages (desktop, dev tools, browser, etc.)
  4. Installing packages from the repo
  5. Configuring the system (hostname, users, timezone)
  6. Installing GRUB
  7. Rebooting into SwellOS
